# グラフごとの正しい解釈

作成日: 2026-07-06 JST

## このドキュメントの目的

Bike Sharing Dataset の `cnt` を可視化するときに、

```text
どのグラフで何を見ているのか
そのグラフから何を言ってよいのか
逆に、何を言ってはいけないのか
```

を整理する。

特に重要なのは、**グラフはデータそのものではなく、データの一部の見せ方**という点。

同じ `cnt` でも、月平均、日次合計、時間帯平均、ヒストグラムでは、見ているものが違う。

---

## 1. まず意識すること

グラフを見る前に、次を確認する。

| 確認すること | 理由 |
|---|---|
| 何を集計したか | 平均、合計、中央値では意味が違う |
| どの単位で集計したか | 時間、日、月、曜日、時間帯で意味が違う |
| 横軸は連続か離散か | 折れ線で結んでよいかに関わる |
| 点と点の間に実データがあるか | 線の途中を解釈してよいかに関わる |
| 正確な値を読みたいのか、傾向を見たいのか | 表、棒、折れ線の選択に関わる |

Statistics Canada の可視化ガイドでも、グラフ作成前に「どんなデータか」「誰に向けるか」「何を伝えたいか」を確認することが重要だと説明している。

今回のEDAでは、まず次の問いを分ける。

```text
1. cntはどれくらいの規模か
2. cntはどの範囲に多いか
3. cntは時間とともに増減しているか
4. cntは何時に多いか
5. cntは曜日・天気・季節で違うか
```

---

## 2. 折れ線グラフ

### 何を見るグラフか

折れ線グラフは、**連続した順序を持つ横軸に沿って、値がどう変化するか**を見るためのグラフ。

Bike Sharing では、次のような用途に向いている。

| データ | 見ること |
|---|---|
| 1時間ごとの `cnt` | 短期間の細かい増減 |
| 日ごとの合計 `cnt` | 長期的な需要の増減 |
| 月ごとの平均 `cnt` | 季節性や年をまたいだ傾向 |

### 読んでよいこと

| グラフ | 読んでよいこと |
|---|---|
| 1時間ごとの折れ線 | その期間内で、どの時間に増減したか |
| 日次集計の折れ線 | 日ごとの需要が上がっているか、下がっているか |
| 月平均の折れ線 | 月単位で見たとき、平均利用数が増えているか、減っているか |

### 読んではいけないこと

月ごとの平均 `cnt` を折れ線で結んだ場合、線の途中に実測値があるわけではない。

例:

```text
2011-01 の平均 cnt
2011-02 の平均 cnt
```

この2点を線で結んでも、線の途中は「1月から2月へ平均値がなめらかに変化した」という実データではない。

線はあくまで、隣り合う集計値の推移を見やすくする補助。

```text
言ってよい:
1月平均より2月平均の方が高い

言ってはいけない:
1月15日から2月15日にかけて、この線の通りにcntが増えた
```

Atlassian の折れ線グラフガイドでも、折れ線は時系列など連続的な横軸で変化パターンを見るためのものだが、測定間隔の選び方によりノイズや傾向の見え方が変わると説明している。また、点の間を滑らかな曲線として補間すると、存在しないデータがあるように見えるため注意が必要とされている。

### Bike Sharing での注意

`hour.csv` を折れ線にするとき、横軸を `dteday` だけにすると不自然になりやすい。

理由:

```text
同じdtedayに0時から23時まで複数行あるため
```

時間単位で見るなら、`dteday + hr` で作った日時を横軸にする。

長期傾向を見るなら、日次や月次に集計してから見る。

## 3. 棒グラフ

### 何を見るグラフか

棒グラフは、**カテゴリや離散的な期間ごとの値を比較する**ためのグラフ。

University of Missouri の可視化ガイドでも、棒グラフはグループ間の値比較に向いており、学期・四半期・年度のような離散期間の比較にも使えると整理されている。

Bike Sharing では、次のような用途に向いている。

| データ | 見ること |
|---|---|
| 月ごとの平均 `cnt` | 各月の平均利用数の比較 |
| 曜日ごとの平均 `cnt` | 曜日による違い |
| 天気カテゴリごとの平均 `cnt` | 天気による違い |
| 時間帯ごとの平均 `cnt` | 0時から23時の需要パターン |

### 読んでよいこと

```text
8月の平均cntは2月より高い
平日と休日で平均cntが違う
晴れの日の平均cntは悪天候の日より高い
```

### 読んではいけないこと

棒グラフは、棒と棒の間に連続的な変化があることを示さない。

例えば、月ごとの棒グラフでは、

```text
1月
2月
3月
```

という独立した集計値を比べている。

```text
言ってよい:
3月の平均cntは2月より高い

言ってはいけない:
2月中旬から3月中旬にかけて直線的に増えた
```

### 月平均では棒グラフが分かりやすい理由

月平均は、各月に1つだけ値がある集計データ。

そのため、正確に比較したいなら折れ線より棒グラフの方が誤解が少ない。

| 目的 | 向いているグラフ |
|---|---|
| 月平均の推移をざっくり見る | 折れ線 |
| 各月の平均値を独立に比較する | 棒 |
| 正確な値を確認する | 表 |

## 4. 点グラフ

### 何を見るグラフか

点グラフは、**各集計値を点として見せる**グラフ。

棒の面積や折れ線のつながりを使わず、点の位置だけで値を見る。

Bike Sharing では、次のような用途に向いている。

| データ | 見ること |
|---|---|
| 月ごとの平均 `cnt` | 各月の値だけを見る |
| 曜日ごとの平均 `cnt` | 曜日の比較 |
| 天気ごとの平均 `cnt` | 天気カテゴリの比較 |

### 読んでよいこと

```text
各月の平均cntがどの位置にあるか
どのカテゴリが高いか、低いか
```

### 読んではいけないこと

点と点の間に値があるとは言えない。

点グラフは、折れ線よりも「点だけが観測・集計された値」という意味を保ちやすい。

---

## 5. ヒストグラム

### 何を見るグラフか

ヒストグラムは、**数値データがどの範囲にどれくらいあるか**を見るグラフ。

University of Missouri のガイドでも、ヒストグラムは数値分布の形、広がり、密度を理解するためのものと説明されている。

Bike Sharing では、`cnt` の分布を見るのに使う。

### 読んでよいこと

```text
cntが小さい時間帯が多い
cntが大きい時間帯は少ない
分布が右に長い
極端に利用数が多い時間帯がある
```

### 読んではいけないこと

ヒストグラムは時系列ではない。

そのため、

```text
いつ増えたか
どの月に多いか
何時に多いか
```

は分からない。

また、ビン幅によって見え方が変わる。

```text
ビンが粗すぎる -> 細かい偏りが見えない
ビンが細かすぎる -> ノイズが目立つ
```

### Bike Sharing での使いどころ

Step 1では、`cnt` の規模感と偏りを見るために使う。

例えば、平均値だけを見ると分からない次の点を確認できる。

```text
少ないcntの時間帯が多いのか
高いcntが一部だけなのか
分布が左右対称か、偏っているか
```

## 6. 箱ひげ図

### 何を見るグラフか

箱ひげ図は、**中央値、四分位、外れ値、ばらつき**を見るためのグラフ。

Bike Sharing では、カテゴリごとの `cnt` の分布比較に向いている。

| 比較 | 見ること |
|---|---|
| 曜日ごとの `cnt` | 曜日で中央値やばらつきが違うか |
| 時間帯ごとの `cnt` | 何時台のばらつきが大きいか |
| 天気ごとの `cnt` | 悪天候では利用数が低くまとまるか |

### 読んでよいこと

```text
この時間帯は中央値が高い
この曜日はばらつきが大きい
この天気カテゴリでは外れ値が多い
```

### 読んではいけないこと

箱ひげ図は、時系列の順序を直接示さない。

例えば、曜日別の箱ひげ図を見ても、

```text
月曜から火曜にかけて増えた
```

とは言えない。

あくまで、曜日カテゴリごとの分布比較。

---

## 7. 散布図

### 何を見るグラフか

散布図は、**2つの数値変数の関係**を見るためのグラフ。

University of Missouri のガイドでも、散布図は2つの数値変数の関係、クラスター、傾向、外れ値を見つけるために使うとされている。

Bike Sharing では、次のような用途に向いている。

| x軸 | y軸 | 見ること |
|---|---|---|
| `temp` | `cnt` | 気温が高いほど利用が増えるか |
| `hum` | `cnt` | 湿度が高いほど利用が減るか |
| `windspeed` | `cnt` | 風速と利用数の関係 |

### 読んでよいこと

```text
気温が高いほどcntが増える傾向がある
湿度が高い領域ではcntが低い点が多い
外れ値がある
```

### 読んではいけないこと

散布図だけでは因果関係は言えない。

```text
言ってよい:
気温とcntには正の関係がありそう

言ってはいけない:
気温が上がったから必ずcntが増えた
```

他の要因も混ざる。

```text
季節
曜日
時刻
天気
年
```

散布図は仮説作りに使い、モデルや追加分析で確認する。

---

## 8. ヒートマップ

### 何を見るグラフか

ヒートマップは、**2つの軸の組み合わせごとの強さ**を見るためのグラフ。

University of Missouri のガイドでも、ヒートマップは2次元上の強度や密度のパターンを見つけるのに役立つとされている。

Bike Sharing では、かなり有用。

| 縦軸 | 横軸 | 色 |
|---|---|---|
| 曜日 | 時間帯 | 平均 `cnt` |
| 月 | 時間帯 | 平均 `cnt` |
| 天気 | 時間帯 | 平均 `cnt` |

### 読んでよいこと

```text
平日の朝夕にcntが高い
休日の昼にcntが高い
冬の深夜はcntが低い
```

### 読んではいけないこと

色は正確な値の読み取りには向かない。

```text
濃い場所は高い
薄い場所は低い
```

というパターンを見るのに向いている。

正確な値を知りたい場合は、表やラベル付きの値も確認する。

---

## 9. 表

### 何を見るものか

表は、**正確な値を確認する**ための形式。

University of Missouri のガイドでも、正確な比較や値の検索が必要な場合は表が向いていると整理されている。

Bike Sharing では、集計結果の確認に使う。

| 表 | 見ること |
|---|---|
| 月ごとの平均 `cnt` | 各月の正確な平均値 |
| 時間帯ごとの平均 `cnt` | 0時から23時の正確な平均値 |
| 曜日ごとの平均 `cnt` | 曜日ごとの正確な比較 |

### グラフとの使い分け

| 目的 | 向いている形式 |
|---|---|
| 傾向を早く掴む | グラフ |
| 正確な値を読む | 表 |
| 外れ値や偏りを探す | グラフ |
| 結果を記録する | 表 |

---

## 10. Bike Sharing の `cnt` で最初に見るもの

Step 1 では、次の順番で十分。

| 順番 | 見るもの | グラフ | 解釈 |
|---:|---|---|---|
| 1 | `cnt` の基本統計量 | 表 | 規模感を見る |
| 2 | `cnt` の分布 | ヒストグラム | 値の偏りを見る |
| 3 | 月ごとの平均 `cnt` | 棒または折れ線 | 季節性・年をまたぐ変化を見る |
| 4 | 時間帯ごとの平均 `cnt` | 棒または折れ線 | 1日の中の需要パターンを見る |
| 5 | 曜日 × 時間帯の平均 `cnt` | ヒートマップ | 平日・休日と時間帯の組み合わせを見る |

### 月ごとの平均 `cnt`

目的:

```text
季節性と長期トレンドを見る
```

解釈:

```text
各月における1時間あたり平均利用数
```

注意:

```text
折れ線の点と点の間は実測値ではない
棒グラフの方が各月の独立した比較には向いている
```

### 時間帯ごとの平均 `cnt`

目的:

```text
何時に利用が多いかを見る
```

解釈:

```text
0時から23時まで、それぞれの時間帯における平均利用数
```

注意:

```text
時系列ではない
0時、1時、2時...という時間帯カテゴリの比較
```

### 日次または時間単位の時系列 `cnt`

目的:

```text
時間の流れに沿った増減を見る
```

解釈:

```text
いつ増えたか、いつ減ったか
```

注意:

```text
hour.csvをそのまま全期間で描くと見づらい
短期間に絞るか、日次・月次に集計する
```

## 11. グラフ選択の早見表

| 見たいこと | 向いているグラフ | Bike Sharingでの例 |
|---|---|---|
| 時間の流れに沿った変化 | 折れ線 | 日次合計、月平均、1週間の時間別 `cnt` |
| 月や曜日などの比較 | 棒 | 月別平均、曜日別平均 |
| 値の偏り | ヒストグラム | `cnt` の分布 |
| 中央値・ばらつき・外れ値 | 箱ひげ図 | 時間帯別 `cnt` 分布 |
| 2変数の関係 | 散布図 | `temp` と `cnt` |
| 2軸の組み合わせパターン | ヒートマップ | 曜日 × 時間帯の平均 `cnt` |
| 正確な値 | 表 | 月別平均値一覧 |

## 12. よくある誤解

### 12.1 月平均の折れ線を実測時系列だと思う

誤り:

```text
月平均の点と点の間にも、線の通りにcntが存在する
```

正しい理解:

```text
線は月ごとの集計値をつないだ補助線
途中の値は実測値ではない
```

### 12.2 時間帯平均を時系列だと思う

誤り:

```text
0時から23時にかけて、1日の中で必ずこの通りに変化する
```

正しい理解:

```text
全期間の同じ時間帯をまとめた平均
実際の1日ごとの動きとは違う
```

### 12.3 ヒストグラムから「いつ」を読もうとする

誤り:

```text
ヒストグラムで高いcntがあるから、夏に多い
```

正しい理解:

```text
ヒストグラムは値の分布を見る
いつ多いかは時系列や月別集計を見る
```

### 12.4 散布図から因果を断定する

誤り:

```text
気温が高いからcntが増えた
```

正しい理解:

```text
気温とcntに関係がありそう
ただし、曜日・季節・時刻なども影響している可能性がある
```

## 13. 今回の学習での結論

グラフを見るときは、次のように考える。

```text
このグラフは何を集計したものか
横軸は連続か、カテゴリか
点と点の間に意味はあるか
正確な値を読むべきか、傾向を見るべきか
このグラフだけで言えることと言えないことは何か
```

特に、月平均や時間帯平均のような集計データでは、

```text
集計値そのもの
集計値を線や棒で表現した見た目
```

を分けて考える。

これができると、EDAで見つけた傾向をそのまま思い込みにせず、次の特徴量作成やモデル評価につなげやすくなる。
